# Path Traversal - File Path Traversal, Simple Case

## Overview

**Lab:** File path traversal, simple case  
**Platform:** PortSwigger Web Security Academy  
**Difficulty:** APPRENTICE  
**Category:** Path Traversal

## Objective

This lab contains a path traversal vulnerability in the display of product images.

To solve the lab, retrieve the contents of the /etc/passwd file.

## Reconnaissance

### Initial Analysis

The application displays product images by fetching them through a request that includes a filename parameter. This parameter is vulnerable to path traversal attacks.

### Target Endpoint

```
https://0a540095030cf46380783f8300ed0079.web-security-academy.net/
```

## Exploitation

### Step 1: Identify the Vulnerable Parameter

Using browser DevTools (Network tab), I intercepted the request that fetches product images. The request contains a `filename` parameter that specifies which image file to load.

### Step 2: Modify the Filename Parameter

I modified the `filename` parameter to include a path traversal sequence to access the `/etc/passwd` file:

**Payload:**

```
../../../etc/passwd
```

**Exploit URL:**

```
https://0a540095030cf46380783f8300ed0079.web-security-academy.net/image?filename=../../../etc/passwd
```

The `../` sequences navigate up the directory structure, allowing access to files outside the intended directory.

## Key Takeaways

- Path traversal vulnerabilities occur when user input is used to construct file paths without proper validation
- The `../` sequence allows navigating up directory levels to access files outside the intended directory
- Browser DevTools can be used effectively for intercepting and modifying requests during security testing
